## API Endpoints

```python
@router.get("/api/detachments")
async def list_detachments(faction: Optional[str] = None):
    """Вернуть список детачментов. С фильтрацией по faction."""
    dets = registry.list_detachments(faction=faction)
    return [{"name": d.name, "faction": d.faction, "description": d.description[:100]}
            for d in dets]


@router.get("/api/detachments/{detachment_name}")
async def detachment_detail(detachment_name: str):
    """Вернуть полные данные детачмента: правила, стратагемы, энхансменты. """
    det = registry.get_detachment(detachment_name)
    if not det:
        raise HTTPException(404)
    return {
        "name": det.name,
        "faction": det.faction,
        "description": det.description,
        "detachment_rule": det.detachment_rule,
        "stratagems": det.stratagems,       # [{name, cost, when, effect}]
        "enhancements": det.enhancements,   # [{name, points, effect}]
    }
```

## Alpine.js Controller

```javascript
// web/static/detachment_picker.js
function detachmentPicker() {
    return {
        detachments: [],
        selectedDetachment: null,
        loading: false,
        error: null,

        init() {
            // Listen for faction changes from parent
            document.addEventListener('faction-changed', (e) => {
                this.loadDetachments(e.detail.faction);
            });
        },

        loadDetachments(faction) {
            if (!faction) { this.detachments = []; return; }
            this.loading = true;
            this.error = null;
            fetch(`/api/detachments?faction=${faction}`)
                .then(r => r.json())
                .then(data => {
                    this.detachments = data;
                    this.loading = false;
                })
                .catch(err => {
                    this.error = 'Failed to load detachments';
                    this.loading = false;
                });
        },

        selectDetachment(name) {
            this.loading = true;
            fetch(`/api/detachments/${encodeURIComponent(name)}`)
                .then(r => r.json())
                .then(data => {
                    this.selectedDetachment = data;
                    this.loading = false;
                })
                .catch(() => { this.loading = false; });
        },

        confirmDetachment() {
            // Emit event to parent team builder
            const event = new CustomEvent('detachment-selected', {
                detail: { detachment: this.selectedDetachment.name }
            });
            document.dispatchEvent(event);
        },
    };
}
```

## State Handling

| Состояние | Поведение |
|-----------|-----------|
| **Нет фракции** | Серый текст "Select a faction first" |
| **Загрузка списка** | Спиннер + skeleton-строки (3 серых блока) |
| **Нет детачментов** | "No detachments found for this faction" |
| **Выбор детачмента** | Карточка с правилами, стратагемами, энхансментами |
| **Ошибка API** | Красная плашка "Failed to load", кнопка Retry |

## Implementation Note — 2026-05-09

### Compact informative picker

The Detachment Picker was redesigned because the previous screen was visually too large and still did not surface enough useful information before selection.

**Active file:** `web/templates/partials/detachment_picker.html`

Current UX:

- Detachments render as compact summary cards instead of tall full-detail blocks.
- Cards use a two-column grid on medium+ screens and a single column on narrow screens.
- Each card shows the detachment name, detachment rule name, a short rule preview, and count badges for stratagems and enhancements.
- The selected card remains visually distinct through border/ring state.
- Full rule text, stratagems, and enhancements remain available in the detail panel after selection; the panel is scrollable so the picker no longer pushes the whole Team Builder page down.

### Data contract

The compact cards depend on summary fields from the detachment API/list model:

```json
{
  "name": "War Horde",
  "faction": "orks",
  "rule_name": "Get Stuck In",
  "rule_description": "Melee weapons gain Sustained Hits 1...",
  "stratagem_count": 6,
  "enhancement_count": 4
}
```

The full detail view still uses the existing detail payload with `detachment_rule`, `stratagems`, and `enhancements`.
