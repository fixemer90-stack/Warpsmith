# F4.8: SVG Icons Integration in Unit Cards

**Файлы:** `web/static/icons/*.svg` (16 файлов — уже есть от Phase 0.8),
`web/templates/partials/unit_card.html` (создать),
`backend/loader/icon_map.py` (расширение — добавить пути и CSS-class helper),
`web/static/unit_card.css` (создать)
**Зависимости:** Phase 0.8 (SVG Icons + ICON_MAP), F4.1 (Faction Browser),
F4.2 (Unit Modal)
**Часы:** 2h

**Цель (SMART):** Интегрировать существующие 16 SVG-иконок в карточки юнитов,
весь Team Builder и модалку. Критерий: каждая карточка юнита показывает SVG-иконку
категории, цвет рамки соответствует CATEGORY_COLORS, hover-эффект увеличивает иконку
на 10%, inline SVG (не external image) для минимальной загрузки. 2 часа.

## Icon Map — Extended

```python
# backend/loader/icon_map.py — расширение
from dataclasses import dataclass
from typing import Optional

@dataclass
class IconInfo:
    filename: str
    label: str
    color: str
    svg_content: Optional[str] = None  # кэшированный inline SVG
    order: int = 0

# Добавить в ICON_MAP: filename → IconInfo
ICON_REGISTRY: dict[str, IconInfo] = {}
for cat, filename in ICON_MAP.items():
    ICON_REGISTRY[cat] = IconInfo(
        filename=filename,
        label=CATEGORY_LABELS.get(cat, cat),
        color=CATEGORY_COLORS.get(cat, "#6b7280"),
        order=CATEGORY_ORDER.index(cat) if cat in CATEGORY_ORDER else 99,
    )


def get_icon_html(category: str, size: int = 24, class_name: str = "") -> str:
    """
    Вернуть inline SVG как HTML-строку.
    Используется в Jinja2 шаблонах через {% raw %} или как safe string.
    """
    info = ICON_REGISTRY.get(category, ICON_REGISTRY.get("infantry"))
    if not info:
        return ""

    svg_path = f"web/static/icons/{info.filename}"
    try:
        with open(svg_path) as f:
            svg = f.read()
    except FileNotFoundError:
        return f'<!-- icon {info.filename} not found -->'

    # Вставить размер и класс
    svg = svg.replace('<svg ', f'<svg width="{size}" height="{size}" class="{class_name}" ')
    return svg


def get_card_style(category: str) -> str:
    """Вернуть CSS-стиль для карточки юнита по категории."""
    info = ICON_REGISTRY.get(category, ICON_REGISTRY.get("infantry"))
    return f"border-left: 3px solid {info.color};"
```

```javascript
// В team_builder.js — helper для inline SVG
function getIconHtml(category, size = 24) {
    // Карта категория → SVG-контент (загружается с сервера при старте)
    const svgMap = window._iconSvgMap || {};
    const svg = svgMap[category] || svgMap['infantry'] || '';
    if (!svg) return '';
    return svg.replace('<svg ', `<svg width="${size}" height="${size}" style="fill: currentColor" `);
}
```

## Server-Side Inline SVG Injection

```python
# web/routes/pages.py — передать SVG-карту на фронт

@router.get("/team-builder")
async def team_builder_page(request: Request):
    """Рендер Team Builder с inline SVG картой."""
    # Загрузить все SVG в словарь
    import os
    import glob

    svg_dir = "web/static/icons"
    svg_map = {}
    for cat, filename in ICON_MAP.items():
        path = os.path.join(svg_dir, filename)
        if os.path.exists(path):
            with open(path) as f:
                svg_map[cat] = f.read()

    return templates.TemplateResponse("team_builder.html", {
        "request": request,
        "factions": registry.list_factions(),
        "categories": CATEGORY_ORDER,
        "icon_svg_map": svg_map,       # для Jinja2 инлайн
        "_icon_svg_map_json": json.dumps(svg_map),  # для Alpine.js
    })
```

```html

</script>
```

## Edge Cases

| Ситуация | Поведение |
|----------|-----------|
| SVG файл не найден | Отображается fallback-иконка (infantry.svg), лог ошибки |
| Неизвестная категория | Используется infantry.svg по умолчанию |
| Hover на карточке | Иконка scale(1.15), плавный transition 0.2s |
| Очень длинное название юнита | text-truncate (CSS), иконка не сжимается |
| Touch-устройства | hover не срабатывает — иконка всегда нормального размера |
