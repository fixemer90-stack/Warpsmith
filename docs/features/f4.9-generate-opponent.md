# F4.9: Generate Random Opponent — AI Roster Generator

**Файлы:** `web/routes/api_rosters.py` (`POST /api/rosters/generate`),
`web/static/scenario_setup.js` (кнопка "Generate Random Opponent")
**Зависимости:** F2.10 (Roster CRUD), F3.2 (Faction AI Profiles)
**Часы:** 2h

**Цель (SMART):** Реализовать кнопку "Generate Random Opponent" на странице Scenario Setup,
которая создаёт валидный AI-ростер за 1 клик: выбирает случайную фракцию, наполняет юнитами
в рамках PTS-бюджета, назначает Warlord, соблюдает caps (3× на юнит, 1× Epic Hero), возвращает
готовый ростер для использования как `roster_b` в симуляции. 2 часа.

---

## API Endpoint

```python
# POST /api/rosters/generate
# Request (optional):
# {
#   "faction": "orks" | null (random if null),
#   "pts_limit": 2000
# }
#
# Response:
# {
#   "id": "generated_<timestamp>",
#   "faction": "orks",
#   "name": "Random Ork Waaagh!",
#   "total_pts": 1995,
#   "units": [
#     {"name": "Warboss", "squad_size": 1, "pts": 80, "is_warlord": true},
#     {"name": "Boyz", "squad_size": 10, "pts": 170},
#     ...
#   ],
#   "warlord_unit_id": "..."
# }
```

## Edge Cases

| Ситуация | Поведение |
|----------|-----------|
| Фракция не указана | Выбрать случайную из доступных |
| Недостаточно PTS для минимального ростера | Вернуть ошибку, предложить увеличить PTS |
| Все Epic Hero заняты (1× cap) | Не добавлять больше Epic Hero |
| Юнит без PTS в вики | Пропустить (не добавлять в ростер) |

## Тесты

```python
def test_generate_roster_random_faction(client):
    resp = client.post("/api/rosters/generate", json={})
    assert resp.status_code == 200
    data = resp.json()
    assert data["faction"] in ["orks", "tau", "adeptus-mechanicus"]
    assert len(data["units"]) > 0
    assert data["total_pts"] <= 2000

def test_generate_roster_specific_faction(client):
    resp = client.post("/api/rosters/generate", json={"faction": "orks"})
    assert resp.status_code == 200
    assert resp.json()["faction"] == "orks"

def test_generate_roster_warlord_assigned(client):
    resp = client.post("/api/rosters/generate", json={"faction": "orks"})
    units = resp.json()["units"]
    warlords = [u for u in units if u.get("is_warlord")]
    assert len(warlords) == 1

def test_generate_roster_pts_limit(client):
    resp = client.post("/api/rosters/generate", json={"pts_limit": 500})
    assert resp.json()["total_pts"] <= 500
```

## Implementation Note — 2026-05-09

### Generated roster validity contract

Generated armies must be immediately saveable and playable. The generator now treats validity as a hard requirement, not as a best-effort random list.

**Active files:**

- `web/routes/api_rosters.py` — `POST /api/rosters/generate`
- `web/static/scenario_setup.js` — generated-opponent flow and auto-play redirect
- `tests/test_generate_roster.py` — generator regression coverage

Rules enforced by the generator:

1. **Exactly one Warlord** — one generated unit has `is_warlord: true`; all others are false/omitted.
2. **Correct squad size source** — minimum generated model count comes from `unit.squad_size["min"]`, not `unit.model_count`. This fixed the invalid `Nobz: min 2 models, got 1` case.
3. **PTS formula matches backend validation** — generated unit `pts` uses the same per-model formula as roster validation, so the generated total is not accepted in UI but rejected by API.
4. **Warlord fits inside points cap** — if random picks fill the limit before a Warlord is present, the generator removes non-Warlord picks until the cheapest Warlord-capable unit fits.
5. **Simulation redirect is robust** — Scenario Setup reads the top-level `/api/auto-play` `game_id` first, then falls back to nested result data, and redirects to `/result/{game_id}`.

### Regression fixed

Before the fix, a generated roster could be returned by `/api/rosters/generate` but fail when saved or simulated because one unit used an invalid squad size. Example failure:

```text
Nobz: min 2 models, got 1
```

The root cause was using `unit.model_count` as the generated squad size source. Validation uses YAML `squad_size`, so the generator now uses the same source.
