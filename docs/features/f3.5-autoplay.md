# F3.5: Auto-play — AI vs AI Full Scenario

**Файл:** `backend/engine/ai/autoplay.py`
**Зависимости:** F3.1 (Decision), F3.2 (Faction AI), F3.4 (Deployment AI), F2.5 (Game Loop)
**Часы:** 6h

**Цель (SMART):** Запустить полную AI vs AI симуляцию: deploy → N раундов → результат. Критерий: `/api/auto-play` возвращает winner, VP, round_logs; replay сохраняется в SQLite. 6 часов.

---

## Реализация

```python
@dataclass
class AutoPlayConfig:
    max_rounds: int = 5
    deployment_type: DeploymentType = DeploymentType.DAWN_OF_WAR
    seed: int = 42

@dataclass
class AutoPlayResult:
    game_state: GameState
    winner: str | None
    reason: str           # "rounds_completed", "army_wiped", "error"
    round_logs: list[dict]
    vp_player1: int
    vp_player2: int
    error: str | None

def run_auto_game(
    roster_a: RosterState,
    roster_b: RosterState,
    mission_name: str = "only_war",
    config: AutoPlayConfig | None = None,
) -> AutoPlayResult:
```

### Поток выполнения

1. `create_empty_game()` — игровое состояние
2. `deploy_game()` — AI расставляет юниты
3. `Scenario(game_state).run_game()` — прогон раундов
4. VP суммируются в `PlayerState.victory_points` через `_command_phase`
5. Результат возвращается как `AutoPlayResult`

## Что НЕ подключено

| Компонент | Где используется | Статус |
|-----------|-----------------|--------|
| `ReplayRecorder` | должен быть в `run_auto_game` | ❌ не вызывается (replay пишется на API-слое) |
| `VPTracker` | должен быть в `_command_phase` | ❌ не вызывается (VP идут напрямую в player.victory_points) |
| `check_end_game` | должен быть в `run_round` | ❌ не вызывается (используется `is_game_over` property) |
