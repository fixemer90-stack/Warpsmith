# F5.1: Dockerfile + docker-compose
**Цель (SMART):** Создать production-ready Dockerfile (multi-stage, slim image) и docker-compose.yml с сервисами app + sqlite-backup. Критерий: `docker compose up` запускает приложение на порту 8000, образ < 200MB, sqlite-данные сохраняются между рестартами через volume. 3 часа.

## Dockerfile

Multi-stage сборка: слой зависимостей кэшируется независимо от кода.

```dockerfile
# === Stage 1: dependencies ===
FROM python:3.12-slim AS builder

WORKDIR /app

# Системные зависимости для numpy
RUN apt-get update && apt-get install -y --no-install-recommends \
    build-essential \
    && rm -rf /var/lib/apt/lists/*

COPY pyproject.toml uv.lock* ./
RUN pip install --no-cache-dir uv && \
    uv pip install --system -r pyproject.toml

# === Stage 2: runtime ===
FROM python:3.12-slim AS runtime

WORKDIR /app

# Только runtime-зависимости (libgcc для numpy)
RUN apt-get update && apt-get install -y --no-install-recommends \
    libgomp1 \
    && rm -rf /var/lib/apt/lists/*

COPY --from=builder /usr/local/lib/python3.12/site-packages /usr/local/lib/python3.12/site-packages
COPY --from=builder /usr/local/bin /usr/local/bin

COPY . .

EXPOSE 8000

HEALTHCHECK --interval=30s --timeout=5s --start-period=10s --retries=3 \
    CMD python -c "import urllib.request; urllib.request.urlopen('http://localhost:8000/')" || exit 1

CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
```

## docker-compose.yml

```yaml
version: "3.9"

services:
  app:
    build: .
    ports:
      - "8000:8000"
    env_file:
      - .env
    environment:
      - HOSTING=true
      - DB_PATH=/data/simulator.db
    volumes:
      - sqlite_data:/data
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "python", "-c", "import urllib.request; urllib.request.urlopen('http://localhost:8000/')"]
      interval: 30s
      timeout: 5s
      retries: 3
      start_period: 10s

volumes:
  sqlite_data:
```

## Использование

```bash
# Сборка и запуск
docker compose build
docker compose up -d

# Логи
docker compose logs -f

# Остановка
docker compose down

# Полная очистка (удалить volume)
docker compose down -v
```

## Образ: размер и безопасность

| Проверка | Ожидание |
|----------|----------|
| Размер образа | < 200MB (slim + multi-stage) |
| root-процесс | Нет — добавить `USER nobody` или создать `appuser` |
| Порты | Только 8000/tcp |
| Секреты | Всё через `.env` / docker secrets |
| Healthcheck | `/` эндпоинт отвечает 200 |
