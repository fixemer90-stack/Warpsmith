# F4.11: Movement Phase — Full 10ed Implementation

**Файл:** `backend/engine/scenario.py`
**Зависимости:** F2.5 (Game Loop), F2.1 (Game State)
**Часы:** 4h

**Цель (SMART):** Реализовать Movement Phase по правилам 10th edition:
Normal Move / Advance / Fall Back / Remain Stationary. Юниты AI распределяются
по objectives равномерно. Критерий: юниты двигаются к objectives → VP > 0
в AI vs AI тесте. 4 часа.

---

## Правила (10ed)

См. `wiki/rules/10th/Movement Phase.md`:

| Действие | Движение | Стрельба | Charge |
|----------|----------|----------|--------|
| **Normal Move** | до M" | ✅ | ✅ |
| **Advance** | M" + D6 | ❌ (кроме Assault) | ❌ |
| **Fall Back** | до M" (только если engaged) | ❌ | ❌ |
| **Remain Stationary** | 0" | ✅ (Heavy без штрафа) | ✅ |

## Реализация

### UnitState: поле `has_advanced`

```python
@dataclass
class UnitState:
    ...
    has_advanced: bool = False  # Не может стрелять/заряжать после Advance/Fall Back
```

Сбрасывается в `next_phase()` при старте нового раунда.

### MovementAction (AI decision)

```python
_pick_movement_action(unit, target, move_stat, own_player_id) → str

Если engaged         → fall_back (mandatory)
Если на objective    → 50% remain_stationary, 50% normal_move
Если далеко (> M")   → 60% advance, 40% normal_move
Если близко (≤ M")   → 70% normal_move, 20% remain_stationary, 10% advance
```

### Objective Distribution

Алгоритм равномерного распределения юнитов по objectives:

1. Для каждого игрока собираются: alive units, objectives (из MissionConfig).
2. Greedy-присвоение: для каждого objective — ближайший незанятый юнит.
3. Неприсвоенные юниты → движутся к ближайшему врагу.
4. Если objectives нет (Purge the Foe) → все к ближайшему врагу.

Гарантирует, что все 3 точки будут удерживаться разными юнитами.

### Movement Toward Target

```python
_move_toward(unit, target, move_cells, own_player_id)
```

- Движение по направляющему вектору к цели
- Остановка за 1 клетку до врага (Engagement Range = 1")
- Проверка bounds, terrain (IMPASSABLE), occupation
- Выбирается лучшая клетка с максимальным прогрессом к цели

### Fall Back

```python
_fall_back(unit, own_player_id, move_stat)
```

- Направление: от конкретного engaging enemy ИЛИ к своей deployment zone
- lower player ("1"/"p1") → y=0, upper ("2"/"p2") → y=map_height-1
- Устанавливает `has_advanced = True`

### Интеграция в фазы

- `_shooting_phase`: проверяет `has_advanced` → пропускает юнит
- `_charge_phase`: проверяет `has_advanced` → пропускает юнит

## Отличия от 9ed

- Нет «наступления» как отдельного типа — Advance встроен как опция
- Нет штрафа -1 к попаданию за движение (пехота)
- Транспорт/высадка не реализованы (за пределами скоупа)
- Deep Strike / Strategic Reserves не реализованы (за пределами скоупа)
