**Цель (SMART):** Реализовать полный цикл раунда: 6 фаз (Command -> Movement -> Shooting -> Charge -> Fight -> Morale), каждая вызывает соответствующие методы engine. ## Core Loop

The actual implementation uses a `Scenario` class (not top-level functions). The spec below reflects that pattern:

```python
def run_round(state: GameState, map_: GameMap,
              mission: Mission, log: RoundLog) -> GameState:
    """Пройти один полный раунд игры.

    Note: this is a functional wrapper for testing. The actual Scenario class
    manages state internally — see backend/engine/scenario.py.
    """

    # Command Phase — оба игрока
    state = command_phase(state, mission, log)

    # Determine turn order via command_priority
    # Player without priority goes first in Movement/Shooting/Charge/Fight
    priority_player = next(pid for pid, p in state.players.items() if p.command_priority)
    first_player = [pid for pid in state.players if pid != priority_player][0]
    second_player = priority_player

    for active_pid in [first_player, second_player]:
        state.active_player = active_pid
        state.current_phase = GamePhase.MOVEMENT
        state = movement_phase(state, map_, log, active_pid)
        state.current_phase = GamePhase.SHOOTING
        state = shooting_phase(state, map_, log, active_pid)
        state.current_phase = GamePhase.CHARGE
        state = charge_phase(state, map_, log, active_pid)
        state.current_phase = GamePhase.FIGHT
        state = fight_phase(state, map_, log, active_pid)

    # Morale Phase (both players simultaneously)
    state.current_phase = GamePhase.MORALE
    state = morale_phase(state, log)

    # End of round — use next_phase() to advance round counter and reset flags
    state.next_phase()

    return state


def run_game(state: GameState, map_: GameMap,
             mission: Mission, max_rounds: int = 5) -> list[RoundLog]:
    """Полная игра: до max_rounds раундов или до game over."""
    logs = []
    for r in range(max_rounds):
        if state.is_game_over:
            break
        log = RoundLog(round=state.current_round)
        state = run_round(state, map_, mission, log)
        logs.append(log)
        # Check end conditions
        if state.is_game_over:
            break
    return logs
```

```python
def shooting_phase(state: GameState, map_: GameMap,
                   log: RoundLog, active_pid: str) -> GameState:
    """Shooting Phase: choose target, make attacks."""
    player = state.players[active_pid]
    # Find opponent
    opponent_pid = [pid for pid in state.players if pid != active_pid][0] if len(state.players) >= 2 else None

    for unit in player.units.values():
        if not unit.is_alive or unit.has_shot or unit.is_engaged:
            continue
        if opponent_pid is None:
            continue
        opponent = state.players[opponent_pid]

        if not targets:

        # Resolve combat

        state.deal_damage(target.unit_id, damage)
        unit.has_shot = True
```

```python
def _check_game_end(state: GameState, mission: Mission) -> bool:
    """Проверить условия окончания игры."""
    # Delegates to state.is_game_over (checks max_rounds + VP threshold)
    if state.is_game_over:
        return True
    # Army wiped check
    for pid, player in state.players.items():
        if all(not u.is_alive for u in player.units.values()):
            return True
    return False
```

## Implementation Notes

1. **`Scenario` class** (`backend/engine/scenario.py`) is the canonical implementation. The functional helpers above (`run_round`, `command_phase`, etc.) serve as spec/test reference and may diverge from the actual `Scenario` methods.

2. **Command Priority** replaces the old `state.turn`:
   - `state.players[pid].command_priority` determines who goes first/last.
   - In Movement/Shooting/Charge/Fight — **non-priority player goes first**.
   - Set via `state._determine_command_priority()` (random round 1, swap thereafter).

3. **Phase transitions** are handled by `state.next_phase()` which:
   - Cycles through `COMMAND → MOVEMENT → SHOOTING → CHARGE → FIGHT → MORALE`
   - After MORALE wraps to COMMAND, increments `current_round`, resets all unit flags (`has_moved`, `has_shot`, `has_charged`, `has_fought`, `is_engaged`, `is_fighting`, `is_battle_shocked`)
   - Calls `_determine_command_priority()` for the new round

4. **No `RosterState`** — players own a dict of `UnitState` objects directly via `PlayerState.units`. Iterate with `player.units.values()`.

5. **Unit flags** are tracked per-unit and reset at round boundary by `next_phase()`:
   - `has_moved` — set after moving in Movement phase
   - `has_shot` — set after resolving shooting (replaces old `is_shooting`)
   - `has_charged` — set after a charge move
   - `has_fought` — set after melee resolution

6. **WIP items** in current codebase:
   - Movement phase: no actual movement logic
   - Shooting phase: no target selection or combat resolution
   - Charge phase: no charge distance rolling or engagement
   - Fight phase: simplified melee (1 damage exchange), no weapon-based resolution
   - Morale phase: basic check exists in Scenario class
   - No AI decision system integrated
   - No test file exists (`tests/test_scenario.py` not yet created)
