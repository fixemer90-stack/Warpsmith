# F2.14: Primary Missions — The Ritual, Supply Drop, Scorched Earth

**Файлы:** `backend/state/mission.py` (расширение), `backend/engine/scenario.py` (фаза Command)
**Зависимости:** F2.4 (Missions — scoring, objectives), F2.1 (Game State)
**Часы:** 12h

**Цель (SMART):** Реализовать 3 новые Primary Mission механики сверх существующих Take and Hold и Purge the Foe:
The Ritual (ритуальные действия для создания objectives), Supply Drop (борьба за маркеры снабжения),
Scorched Earth (захват и «выжигание» objectives). Каждая задаётся через MissionConfig с уникальным
`scoring_rule`, кастомной логикой в Command Phase и новыми типами objectives.
Критерий: 5 миссий в реестре, каждая с уникальной механикой, 3+ теста на миссию. 12 часов.

## Data Model

### Новые типы MissionObjective

```python
class ObjectiveType(Enum):
    STANDARD = "standard"        # обычная точка (Take and Hold)
    RITUAL_SITE = "ritual_site"  # место для ритуала (The Ritual)
    SUPPLY_MARKER = "supply_marker"  # маркер снабжения (Supply Drop)
    SCORCHED = "scorched"        # выжженная точка (Scorched Earth)

@dataclass
class MissionObjective:
    x: int; y: int; label: str
    obj_type: ObjectiveType = ObjectiveType.STANDARD
    controlled_by: str | None = None
    is_contested: bool = False
    is_scorched: bool = False         # Scorched Earth — выжжена
    ritual_progress: int = 0          # The Ritual — 0..3 (3 = complete)
    supply_holder: str | None = None  # Supply Drop — кто держит маркер
```

### MissionConfig — новые поля

```python
@dataclass
class MissionConfig:
    name: str
    deployment: DeploymentType
    description: str
    objectives: list[MissionObjective]
    max_rounds: int = 5
    scoring_rule: str = "standard"
    # Новые поля для Primary Missions:
    ritual_threshold: int = 3          # The Ritual: сколько progress нужно
    supply_drop_rounds: list[int] = field(default_factory=lambda: [2, 4])  # раунды появления маркеров
    scorch_required_rounds: int = 1    # Scorched Earth: сколько раундов удержания для выжигания
```

## Миссия 3: The Ritual (Ритуал)

**Суть:** Игроки выполняют ритуальные действия на специальных точках (Ritual Sites).
При достижении порога ритуала создаётся новый objective marker.

### Механика

1. На карте 3 Ritual Sites (динамически размещаются как обычные objectives)
2. В конце Command Phase: если игрок контролирует Ritual Site, `ritual_progress += 1`
3. Когда `ritual_progress >= ritual_threshold` (3):
   - Ritual Site становится обычным objective (STANDARD)
   - Создаётся новый objective в случайной точке карты (RITUAL_SITE)
4. VP: 3 VP за каждый завершённый ритуал + стандартный scoring за обычные objectives

### Scoring Rule: `ritual`

```python
def score_ritual(mission: Mission) -> dict[int, int]:
    """The Ritual: VP for completed rituals + standard control."""
    vp = {pid: 0 for pid in mission.state.players}
    for obj in mission.config.objectives:
        if obj.obj_type == ObjectiveType.RITUAL_SITE and obj.ritual_progress >= mission.config.ritual_threshold:
            # Completed ritual — award 3 VP to the controller
            if obj.controlled_by:
                vp[obj.controlled_by] += 3
        elif obj.obj_type == ObjectiveType.STANDARD and obj.controlled_by and not obj.is_contested:
            vp[obj.controlled_by] += 1
    return vp
```

### Command Phase — обновление ритуалов

```python
def _update_rituals(mission: Mission) -> None:
    for obj in mission.config.objectives:
        if obj.obj_type != ObjectiveType.RITUAL_SITE:
            continue
        if obj.controlled_by and not obj.is_contested:
            obj.ritual_progress += 1
            if obj.ritual_progress >= mission.config.ritual_threshold:
                # Ritual complete — spawn new objective
                obj.obj_type = ObjectiveType.STANDARD
                obj.label = f"Ritual Site {obj.label} (Complete)"
                _spawn_new_ritual_site(mission)
```

## Миссия 4: Supply Drop (Снабжение)

**Суть:** Специальные маркеры снабжения появляются в заданные раунды.
Игроки борются за контроль над ними. Маркер движется вместе с контролирующим юнитом.

### Механика

1. 3 Supply Markers. Появляются в раундах `supply_drop_rounds` (по умолчанию: раунды 2 и 4)
2. Когда маркер появляется — размещается в центре карты
3. Юнит, находящийся в радиусе 3\" от маркера, может «подобрать» его (становится `supply_holder`)
4. Маркер движется вместе с юнитом-носителем
5. Если носитель уничтожен — маркер падает на его последнюю позицию
6. VP: 3 VP за удержание маркера в конце раунда, 5 VP если удержал до конца игры

### Scoring Rule: `supply_drop`

```python
def score_supply_drop(mission: Mission) -> dict[int, int]:
    """Supply Drop: VP for holding supply markers."""
    vp = {pid: 0 for pid in mission.state.players}
    for obj in mission.config.objectives:
        if obj.obj_type != ObjectiveType.SUPPLY_MARKER:
            continue
        holder = obj.supply_holder
        if holder and not obj.is_contested:
            vp[holder] += 3
    return vp
```

### Command Phase — появление маркеров

```python
def _update_supply_drops(mission: Mission, current_round: int) -> None:
    config = mission.config
    if current_round in config.supply_drop_rounds:
        for obj in config.objectives:
            if obj.obj_type == ObjectiveType.SUPPLY_MARKER and obj.supply_holder is None:
                # Place at center if not yet active
                if obj.x == 0 and obj.y == 0:
                    cx, cy = mission.state.map_width // 2, mission.state.map_height // 2
                    obj.x, obj.y = cx, cy
    # Update marker positions based on holder
    for obj in config.objectives:
        if obj.obj_type == ObjectiveType.SUPPLY_MARKER and obj.supply_holder:
            _move_marker_with_holder(obj, mission)
```

## Миссия 5: Scorched Earth (Выжженная земля)

**Суть:** Игроки захватывают objectives и могут «выжечь» их, лишая врага возможности
набирать с них VP в будущих раундах.

### Механика

1. 5 objectives (как Take and Hold)
2. В конце Command Phase: если игрок контролирует objective, он может начать выжигание
3. Выжигание требует удержания `scorch_required_rounds` (по умолчанию 1) последовательных раундов
4. После выжигания objective помечается `is_scorched = True`
5. Выжженный objective: +2 VP немедленно контролирующему, но больше НЕ даёт VP никому
6. VP: стандартный scoring за невыжженные objectives + бонус за выжигание

### Scoring Rule: `scorched_earth`

```python
def score_scorched_earth(mission: Mission) -> dict[int, int]:
    """Scorched Earth: standard VP for non-scorched + bonus for scorching."""
    vp = {pid: 0 for pid in mission.state.players}
    for obj in mission.config.objectives:
        if obj.is_scorched:
            continue  # No more VP from scorched objectives
        if obj.controlled_by and not obj.is_contested:
            vp[obj.controlled_by] += 1
    return vp
```

### Command Phase — выжигание

```python
def _update_scorched_earth(mission: Mission) -> None:
    for obj in mission.config.objectives:
        if obj.is_scorched:
            continue
        if obj.controlled_by and not obj.is_contested:
            obj.ritual_progress += 1  # reuse field for scorch counter
            if obj.ritual_progress >= mission.config.scorch_required_rounds:
                obj.is_scorched = True
                # Bonus VP for scorching
                mission.state.players[obj.controlled_by].victory_points += 2
```

## Интеграция в Game Loop

`_command_phase()` в `backend/engine/scenario.py`:

```python
def _command_phase(self) -> None:
    # ... existing CP generation, engagement clear, battle-shock ...

    # Mission-specific Command Phase logic
    if self.state.mission:
        rule = self.state.mission.config.scoring_rule

        # Primary Mission mechanics
        if rule == "ritual":
            _update_rituals(self.state.mission)
        elif rule == "supply_drop":
            _update_supply_drops(self.state.mission, self.state.current_round)
        elif rule == "scorched_earth":
            _update_scorched_earth(self.state.mission)

        # VP scoring (все 5 миссий)
        apply_scoring(self.state, self.state.mission, self.vp_tracker)
```
