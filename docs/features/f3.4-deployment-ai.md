# F3.4: Deployment AI — Zone Placement Logic

**Файл:** `backend/engine/ai/deployment.py`
**Зависимости:** F2.2 (2D Map), F2.4 (Missions), F2.1 (Game State)
**Часы:** 3h

**Цель (SMART):** Разместить юниты в deploy zone согласно MissionConfig.deployment. Критерий: юниты в корректных зонах, не накладываются, не выходят за границы. 3 часа.

---

## Реализация

`deploy_game()` в `backend/engine/ai/deployment.py`:

```python
def deploy_game(
    game_state: GameState,
    roster_a: RosterState,
    roster_b: RosterState,
    mission: Mission,
    deployment_type: DeploymentType | None = None,
) -> GameState:
```

1. Получает deploy zones через `mission.get_deployment_zones()`
2. Расставляет warlord в центр зоны
3. Остальные юниты — с отступом от занятых клеток
4. Поддерживает 3 типа деплоя из `DeploymentType`: DAWN_OF_WAR, SEARCH_AND_DESTROY, CRUCIBLE_OF_BATTLE

## Что НЕ подключено

- **Faction AI Profiles (F3.2)**: профили фракций не учитываются при деплое. Деплой чисто позиционный — юниты ставятся в свободные клетки без фракционной логики.
- **Line of Sight (F2.3)**: деплой не проверяет укрытия/террейн приоритетно.
