# F2.4: Missions — Objectives, Scoring, Deployment, End Game

**Файл:** `backend/state/mission.py` (~595 строк)
**Зависимости:** F2.1 (Game State), F2.2 (Map)
**Часы:** 3h

**Цель (SMART):** Реализовать систему миссий с правилами высадки, условиями захвата точек и подсчётом VP. Критерий: 3 готовые миссии (Only War, Purge the Foe, Take and Hold), каждая задаёт deploy зоны, objectives, scoring rule. VP трекается в `VPTracker`, окончание игры — через `check_end_game()`. 3 часа.

## Data Model

### VPTracker
```python
@dataclass
class VPTracker:
    history: dict[int, list[int]]  # {1: [vp_r1, vp_r2, ...], 2: [...]}
    total: dict[int, int]          # {1: total_vp, 2: total_vp}

    def add(player: int, vp: int)       # добавить VP за раунд
    def round_vp(player: int, round_num: int) -> int
    def leader() -> int                 # 1 или 2
    def is_tied() -> bool
    def margin() -> int                 # abs(total[1] - total[2])
    def summary() -> dict               # {player_1: {total, per_round}, player_2: ..., winner, margin}
```

### GameResult
```python
@dataclass
class GameResult:
    winner: str | None     # "1", "2", или None (ничья)
    reason: str            # "rounds_completed", "army_wiped", "vp_cap"
    vp_tracker: VPTracker
    total_rounds: int
    summary: dict
```

### DeploymentType
```python
class DeploymentType(Enum):

```

### MissionObjective
```python
class MissionObjective:
    x: int; y: int; label: str
```

### MissionConfig
```python
@dataclass
class MissionConfig:
    name: str

    description: str
```

### Mission
```python
@dataclass
```

## Scoring — calculate_victory_points()

Единый метод на Mission, вызывается из `_command_phase()` каждый раунд.
Диспатчит по `config.scoring_rule`:

| scoring_rule | Функция | Логика |
|---|---|---|
| `standard` | `score_standard()` | VP = число контролируемых objectives (по 1 VP за каждый) |
| `progressive` | `score_progressive()` | VP = objectives controlled + бонус +2 за контроль больше чем оппонент |
| `kill_points` | `score_kill_points()` | VP = (% уничтоженных очков армии противника) × 100 |

`SCORING_MAP` — dict, мапящий scoring_rule → функцию.

`apply_scoring(state, mission, vp)` — оркестратор: вызывает scorer, конвертирует player_id (str → int: "p1"/"1" → 1), добавляет VP в VPTracker.

## Objective Control — update_objective_control()

Per 10ed: игрок с **наибольшим суммарным OC** в радиусе 3.0 клеток контролирует точку.
Contested только при равном OC.

```python
for obj in config.objectives:
    oc_by_player = {}  # player_id → total_OC
    for player in state.players:
        total_oc = sum(unit.objective_control for alive units within 3.0)
    # highest OC wins; equal OC → contested
```

## Deployment Zones — get_deployment_zones()

Возвращает `dict[player_id, list[(x,y)]]` — все клетки в зоне высадки игрока.

| DeploymentType | Зона |
|---|---|
| DAWN_OF_WAR | Противоположные края (низ/верх), gap посередине |
| SEARCH_AND_DESTROY | Противоположные углы (6×6) |
| CRUCIBLE_OF_BATTLE | Противоположные длинные края |

Глубина зоны: `deployment_zone_rows // 12` клеток (1 клетка = 12 дюймов = 1 фут).

## End Game — check_end_game()

Вызывается каждый раунд. 3 условия:

1. **VP cap**: любой игрок ≥ 100 VP → `reason="vp_cap"`
2. **Army wiped**: у игрока все юниты `models_remaining <= 0` → `reason="army_wiped"`
3. **Max rounds**: `round_num >= max_rounds` → `reason="rounds_completed"`, tie-break через `_resolve_tie()`

`_resolve_tie()`: killed pts → objectives controlled → fallback player 1.

## Вызов из Game Loop

`_command_phase()` в `backend/engine/scenario.py`:

```python
if self.state.mission:
    vp_awarded = self.state.mission.calculate_victory_points()
    for player_id, vp in vp_awarded.items():
        self.state.players[player_id].victory_points += vp
```
