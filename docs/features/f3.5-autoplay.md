# F3.5: Auto-play — AI vs AI Full Scenario

**Файл:** `backend/engine/ai/autoplay.py`
**Зависимости:** F2.5 (Game Loop), F3.1 (Decision Engine), F3.2 (Faction AI Profiles),
F3.4 (Deployment AI), F2.4 (Missions), F2.1 (GameState — актуальный API)
**Deprecates:** f3.2-ork-ai.md, f3.3-tau-ai.md, f3.9-admech-ai.md
**Часы:** 6h (+ 2h на интеграцию с актуальным GameState)

**Цель (SMART):** Реализовать AutoPlay, запускающий полную симуляцию AI vs AI:
deploy units → run 5 rounds → return replay log. AI решения принимаются через
`choose_action_with_faction_ai()` (F3.2), загружающей профили из YAML `ai:` секции
wiki/factions/*.md. 6 часов.

## Разрыв: старый API vs актуальный

| Компонент | Старый API (в spec / autoplay.py) | Актуальный (в коде) |
|-----------|----------------------------------|---------------------|
| `GameState` | `roster_a`, `roster_b`, `seed`, `mission`, `map_width`, `map_height` | `game_id`, `mission_name`, `players: dict[str, PlayerState]`, `terrain_map` |
| `RosterState` | В `game_state.py` (не существовал) | В `roster.py` — `name`, `faction`, `total_pts`, `units` |
| `PlayerState` | ❌ нет | В `game_state.py` — `player_id`, `name`, `faction`, `cp`, `vp`, `units` |
| AI fallback | `DecisionEngine()` (не существовал) | `choose_action` из `decision.py` |
| `BattlefieldMap` | `(width, height)` без terrain | `(width, height, terrain=np.ndarray)` |

## Edge Cases (целевые)

| Edge Case | Ожидаемое поведение |
|-----------|---------------------|
| Ростер невалиден (превышение PTS) | Вернуть InvalidRosterError |
| Оба AI одинаковые (Ork vs Ork) | Работает корректно |
| Юнит без оружия | Пропустить shooting phase |
| Все юниты одной стороны уничтожены | Завершить игру досрочно |
| Timeout превышен | Вернуть частичный результат с error |
| Seed = 0 (random) | Работает, но невоспроизводимо |
| Фракция без `ai:` секции в YAML | Fallback на `choose_action` (F3.1) |
| Ростер с 1 юнитом | Не падать |
