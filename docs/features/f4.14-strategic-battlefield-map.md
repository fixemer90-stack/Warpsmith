# F4.14: Strategic Battlefield Map — mission objectives, roster units, table scale

**Статус:** done  
**Контекст:** F4.5 Canvas map и F4.10 Leaflet map признаны неудачными для Scenario Setup. Новый вариант намеренно минималистичный: без террейн-шума, без внешней Leaflet-зависимости, с фокусом на том, что нужно для старта симуляции.

## Цель

Показать на Scenario Setup понятную 2D-карту, которая отражает:

1. Реальный масштаб стола по выбранному Game Format:
   - 500 pts: 44×30″
   - 1000 pts: 44×44″
   - 2000 pts: 44×60″
   - 3000 pts: 44×90″
2. Objectives, вычисленные из выбранной mission.
3. Юниты из выбранных/сгенерированных ростеров.
4. Deploy zones обеих сторон.
5. Минималистичную легенду и scale ruler.

## Реализация

**Файлы:**

- `web/templates/partials/battlefield_map.html` — SVG-контейнер, legend, summary.
- `web/static/battlefield_map.js` — renderer карты: grid, objectives, deploy zones, units, 12″ ruler.
- `web/static/scenario_setup.js` — интеграция с выбором game format, mission, roster.
- `web/templates/scenario_setup.html` — подключение нового partial/script.
- `tests/test_canvas_map.py` — regression tests для нового варианта карты.

Удалены из активного UI:

- `web/static/canvas_map.js`
- `web/templates/partials/canvas_map.html`
- `web/static/map_view.js`
- `web/templates/partials/map_view.html`
- Leaflet CDN из `base.html`

## Поведение

### Scale

Карта рисуется в SVG с `viewBox`, пропорциональным table size. Визуальная сетка — 6″, major grid — 12″. Внизу есть 12″ ruler с отдельным нижним padding, чтобы подпись не упиралась в рамку SVG.

### Objectives

`battlefieldMap().getMissionObjectives(mission, width, height)`:

- `only-war`: 3 objectives — Center, Flank L, Flank R.
- `take-and-hold`: 5 objectives — Center, Flank L, Flank R, Home A, Home B.
- `purge-the-foe`: 5 objectives — Center, Flank L, Flank R, Home A, Home B.

Координаты масштабируются от `width/height`, поэтому objectives остаются в правильных относительных позициях для 44×30, 44×44, 44×60, 44×90.

### Units

`scenario_setup.js` загружает выбранные ростеры через `/api/rosters/{id}`, получает detail по каждому юниту через `/api/units/{name}/detail`, затем размещает юниты в deploy zone:

- Player 1: левая 20% зона.
- Player 2: правая 20% зона.

На карте юниты отображаются компактными circle markers с инициалами и `<title>` tooltip.

## Тесты

Проверяется:

- `/scenario-setup` содержит `battlefield-map-svg` и `battlefield_map.js`.
- Старые `map_view.js`, `leaflet.js` не подключаются.
- Base template не содержит Leaflet-зависимость.
- JS содержит `initMap`, `showUnits`, `getMissionObjectives`, `drawScaleRuler`.
- Поддержаны mission keys: `only-war`, `take-and-hold`, `purge-the-foe`.
- `/api/map/tiles` продолжает поддерживать dynamic width/height и deploy zones для обратной совместимости тестов.
