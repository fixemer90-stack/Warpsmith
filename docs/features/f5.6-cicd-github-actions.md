# F5.6: CI/CD — GitHub Actions (lint + test + deploy)
**Цель (SMART):** Создать GitHub Actions workflow: (1) lint на ruff + mypy, (2) pytest с coverage, (3) Docker build + push на GHCR, (4) deploy на Railway. Критерий: push в main запускает полный пайплайн за < 5 мин, тесты с coverage > 80%, образ пушится в registry. 4 часа.

## 📂 Файлы

| Файл | Статус | Описание |
|------|--------|----------|
| [`.github/workflows/ci.yml`](../../.github/workflows/ci.yml) | ✅ | Lint + test + Docker build/push |
| [`.github/workflows/deploy.yml`](../../.github/workflows/deploy.yml) | ✅ | Deploy to Railway (активен), Dokku (закомментирован) |
| `tests/test_coverage.py` | ⚪ | Тест порога покрытия (не создан — порог задан в pyproject.toml через `addopts`) |

**Зависимости:**
- [F5.1 Dockerfile](f5.1-dockerfile.md) — Dockerfile для build-стадии
- [F5.2 Deployment](f5.2-deployment.md) — Railway/Dokku настройки
- [F5.5 Logging + Sentry](f5.5-logging-sentry.md) — Sentry DSN передаётся через secrets

## CI workflow (реальный файл)

**[`.github/workflows/ci.yml`](../../.github/workflows/ci.yml)**

3 параллельные джобы:

| Джоба | Триггер | Шаги |
|-------|---------|------|
| `lint` | Всегда | setup-python → pip install → ruff check → ruff format --check → mypy |
| `test` | После lint | setup-python → pip install `-r pyproject.toml` → `pytest --cov=... --cov-report=xml` |
| `docker` | Только main, после test | docker/setup-buildx → GHCR login → build-push (latest + sha) + GHA cache |

**Отличия от spec:**
- Нет `uv` (используется `pip` — проще для CI)
- Codecov upload — [`if: false`] (не настроен)
- Coverage fail-under задан в `pyproject.toml`
- `tests/test_coverage.py` не создан — порог покрытия в addopts

## Deploy workflow (реальный файл)

**[`.github/workflows/deploy.yml`](../../.github/workflows/deploy.yml)**

Запускается после успешного CI на main. Джобы:

| Джоба | Статус | Описание |
|-------|--------|----------|
| `deploy-railway` | ✅ **Активен** | `npm i @railway/cli` → `railway up --service warpsmith` (нужен `RAILWAY_TOKEN`) |
| `deploy-dokku` | 🔴 `if: false` | GitHub Action dokku/github-action (нужны `DOKKU_REMOTE_URL`, `DOKKU_SSH_KEY`) |

## pyproject.toml — coverage порог

```toml
[tool.pytest.ini_options]
testpaths = ["tests"]
addopts = "-v --tb=short --cov-fail-under=80"
```

## GitHub Secrets (для CI/CD)

| Secret | Описание | Статус |
|--------|----------|--------|
| `RAILWAY_TOKEN` | Railway API token — `railway token` в CLI | ✅ Требуется для деплоя |
| `DOKKU_REMOTE_URL` | `ssh://dokku@server:22/warpsmith` | 🔴 Только для Dokku |
| `DOKKU_SSH_KEY` | Приватный ключ SSH для Dokku | 🔴 Только для Dokku |
| `CODECOV_TOKEN` | Codecov upload token | ⚪ Опционально
