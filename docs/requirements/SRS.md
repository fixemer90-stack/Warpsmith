# Software Requirements Specification (SRS)

## 1. Введение

Battle Simulator — веб-приложение для симуляции сценариев Warhammer 40,000 10-й редакции. Пользователь собирает армию через drag-and-drop интерфейс, выбирает миссию и карту, запускает симуляцию. AI-движок разыгрывает обе стороны по правилам игры и выдаёт пораундовый результат.

### 1.1 Глоссарий

| Термин | Значение |
|--------|----------|
| Unit / Отряд | Группа моделей (Boyz, Strike Team) |
| Datasheet | Карточка юнита с M/T/SV/W/LD/OC и оружием |
| Scenario | Полный бой: миссия + карта + две армии + очерёдность |
| Roster | Сохранённый состав армии (юниты + снаряжение + PTS) |
| Replay | Запись всех раундов и событий сценария |
| Game State | Текущее состояние: позиции, раны, статусы, VP, CP |
| Game Loop | Последовательность раундов и фаз |
| CP | Command Points — тратятся на стратагемы |

### 1.2 Целевая платформа

- Веб-приложение (браузер), развёрнуто на хостинге, доступно через интернет
- FastAPI (Python 3.12+) + Jinja2 + HTMX + Alpine.js
- SQLite для persistence
- Wiki vault как источник данных о юнитах
- JWT авторизация: регистрация → вход → cookie httponly + Secure
- CORS: настраивается через ALLOWED_ORIGINS
- bcrypt для хеширования паролей

## 2. Функциональные требования

### 2.1 Team Builder (FR-1)

| ID | Требование |
|----|------------|
| FR-1.1 | Выбор фракции из выпадающего списка (доступные в вики) |
| FR-1.2 | Drag-and-drop юнитов из библиотеки в ростер |
| FR-1.3 | Фильтрация юнитов по категории (Character, Battleline, Vehicle...) |
| FR-1.4 | Выбор размера отряда и снаряжения в модальном окне |
| FR-1.5 | Live-валидация PTS: прогресс-бар, подсветка при превышении |
| FR-1.6 | Валидация: 1 Warlord, ≤ 3 одинаковых не-Battleline юнитов |
| FR-1.7 | Сохранение ростера в БД (имя, фракция, PTS-лимит) |
| FR-1.8 | Загрузка ранее сохранённого ростера из списка |
| FR-1.9 | Выбор детачмента и его правил |

### 2.2 Scenario Setup (FR-2)

| ID | Требование |
|----|------------|
| FR-2.1 | Выбор миссии из предустановленного списка |
| FR-2.2 | Выбор карты: размер, тип террейна |
| FR-2.3 | Canvas-отображение карты: зоны расстановки, метки целей, укрытия |
| FR-2.4 | Выбор первой стороны (First Turn / Roll-off) |
| FR-2.5 | Загрузка второго ростера (выбрать из сохранённых или собрать новый) |

### 2.3 Simulation Run (FR-3)

| ID | Требование |
|----|------------|
| FR-3.1 | Кнопка «Run Simulation» запускает полный сценарий |
| FR-3.2 | Прогресс-бар с текущим раундом и фазой |
| FR-3.3 | Результат: страница с вкладками по раундам |
| FR-3.4 | Каждый раунд: список событий, обновлённая карта, счёт VP |
| FR-3.5 | Возможность пролистать раунды вперёд/назад |
| FR-3.6 | Сохранение реплея в БД |

### 2.4 Game Engine (FR-4)

| ID | Требование |
|----|------------|
| FR-4.1 | Автоматический Deployment: AI расставляет юнитов в зонах |
| FR-4.2 | Полный Game Loop: Command → Movement → Shooting → Charge → Fight |
| FR-4.3 | Симуляция атак: Hit → Wound → Save → Damage с Monte Carlo |
| FR-4.4 | Ключевые слова оружия и модификаторы |
| FR-4.5 | AI-принятие решений: greedy, фракционные приоритеты |

### 2.5 User Profiles & Auth (FR-5)

| ID      | Требование                                                                 |
| ------- | -------------------------------------------------------------------------- |
| FR-5.1  | Регистрация: email + пароль (мин. 6 символов)                              |
| FR-5.2  | Вход: проверка email + password hash (bcrypt)                              |
| FR-5.3  | Выход: удаление JWT-токена из cookies                                      |
| FR-5.4  | JWT-токен: 7 дней, хранится в httponly cookie                              |
| FR-5.5  | Защита API: эндпоинты /api/* проверяют JWT через Depends(get_current_user) |
| FR-5.6  | Гостевой доступ: без авторизации — localStorage + user_id=1                |
| FR-5.7  | Мои ростера: GET /api/rosters — только свои (по user_id из JWT)            |
| FR-5.8  | Мои симуляции: GET /api/replays — по user_id                               |
| FR-5.9  | Публичные ростера: is_public = true — видны всем                           |
| FR-5.10 | Валидация email при регистрации (формат user@domain.tld)                   |
| FR-5.11 | CORS: разрешённые origin'ы через ALLOWED_ORIGINS env                       |
| FR-5.12 | Вход через Google OAuth 2.0 (OpenID Connect)                               |
| FR-5.13 | Вход через VK OAuth 2.0                                                    |
| FR-5.14 | Привязка нескольких OAuth-провайдеров к одному аккаунту                    |
| FR-5.15 | Автоматическое создание аккаунта при первом входе через OAuth              |
| FR-5.16 | Provider Interface: абстрактный класс для подключения новых провайдеров    |

### 2.6 Architecture (FR-6)

| ID | Требование |
|----|------------|
| FR-6.1 | Wiki Loader: парсинг .md → Registry за < 200 мс |
| FR-6.2 | SQLite: ростера, сценарии, реплеи |
| FR-6.3 | pickle-кэш для быстрого перезапуска без перепарсивания вики |
| FR-6.4 | Все компоненты — чистые функции от Game State (тестируемость) |

### 2.7 Subscription & Billing (FR-7)

| ID | Требование |
|----|------------|
| FR-7.1 | Free tier: 1 сохранённый ростер, реклама, базовый AI |
| FR-7.2 | Premium tier: безлимитные ростера, полный AI, без рекламы |
| FR-7.3 | Stripe Checkout Session — оплата подписки |
| FR-7.4 | Stripe Webhook — обновление tier при успешном платеже |
| FR-7.5 | Stripe Customer Portal — управление/отмена подписки |
| FR-7.6 | Feature Gate: UserFeatures.for_user() — централизованная проверка прав |
| FR-7.7 | Реклама: серверная вставка через Alpine.js (Free) |
| FR-7.8 | Экспорт CSV/JSON — только Premium |
| FR-7.9 | Публичные ростера — Free только просмотр, Premium + создание |
| FR-7.10 | Автоматический downgrade до Free при отмене/окончании подписки |

## 3. Нефункциональные требования

| ID | Требование |
|----|------------|
| NFR-1 | Загрузка страницы Team Builder < 500 мс |
| NFR-2 | Симуляция полного сценария < 30 сек |
| NFR-3 | Покрытие тестами боевого движка > 80% |
| NFR-4 | Имена файлов вики = title: (Obsidian-совместимость) |
| NFR-5 | Веб-интерфейс работает в Chrome, Firefox, Edge |
