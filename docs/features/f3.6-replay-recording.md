# F3.6: Replay Recording — JSON Event Log

**Файл:** `backend/engine/replay.py`
**Зависимости:** F3.5 (Auto-play), F2.1 (Game State)
**Часы:** 3h

**Цель (SMART):** Сохранять результат авто-игры в SQLite для последующего просмотра. Критерий: replay сохраняется по game_id, загружается через `/api/replays/{game_id}`, round_viewer отображает пошагово. 3 часа.

---

## Текущая реализация

### Replay data model (`replay.py`)

```python
@dataclass
class ReplayEvent:    # одно событие (move/shoot/charge/fight/damage/death)
@dataclass
class ReplayRound:    # события одного раунда
@dataclass
class Replay:          # полный реплей игры (список раундов)
@dataclass
class ReplayRecorder:  # сборщик событий (не используется — см. ниже)
```

### Сохранение — на уровне API, не в engine

`POST /api/auto-play` (в `web/routes/api.py`):
1. Вызывает `run_auto_game()` → получает `result`
2. Вручную формирует `Replay` из `result.round_logs`
3. Сохраняет через `save_replay(db, replay)`

**ReplayRecorder НЕ вызывается** из `autoplay.py`. Запись реплея происходит пост-фактум на API-слое, а не per-round/per-phase внутри game loop.

### Загрузка

```python
load_replay(db, game_id) → Replay | None
```

### SQLite

```sql
CREATE TABLE replays (
    game_id TEXT PRIMARY KEY,
    created_at TEXT, roster_a TEXT, roster_b TEXT,
    mission TEXT, deployment TEXT, seed INTEGER,
    replay_json TEXT, summary TEXT, user_id INTEGER
)
```
